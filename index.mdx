## What is Body Works API?

Body Works API is a RESTful service that gives developers access to a large fitness database. It powers workout apps, exercise trackers, and health platforms with structured data on:

- **Exercises** — detailed records with target muscles, equipment, instructional blogs, images, GIFs, and YouTube videos
- **Workout Routines** — curated programs categorized by goal, training level, duration, days per week, and equipment
- **Body Parts** — categorized body regions with exercise counts for filtering
- **Target Muscles** — specific muscle groups with exercise counts
- **Equipment** — gym and bodyweight equipment types with exercise counts

## Base URL

All requests are made to:

```
https://api.bodyworks.akshatjaiswal.me/api/v1
```

## Key features

<AccordionGroup>
  <Accordion title="Rich exercise data">
    Every exercise record includes a name, target muscle, body part, required equipment, step-by-step instructional blog, multiple images, an animated GIF, YouTube video links, and searchable keywords.
  </Accordion>
  <Accordion title="Powerful filtering">
    Filter exercises by body part, target muscle, equipment, or free-text search. Filter routines by goal, workout type, training level, duration, days per week, time per workout, equipment, gender, and category.
  </Accordion>
  <Accordion title="Paginated responses">
    All list endpoints support `page` and `limit` query parameters. Responses include `totalPages` so you can build paginated UIs easily.
  </Accordion>
  <Accordion title="Tiered rate limiting">
    General endpoints allow 20,000 requests per 15-minute window. The routines endpoint uses a stricter limit of 20 requests per 15-minute window to protect data-intensive queries.
  </Accordion>
  <Accordion title="Security-hardened">
    The API uses Helmet for HTTP security headers, enforces HSTS with a 1-year max-age, and restricts CORS origins in production.
  </Accordion>
</AccordionGroup>
