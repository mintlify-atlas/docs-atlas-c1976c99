**`GET /api/v1/equipments`**

Returns all equipment types available in the exercise library. This endpoint takes no parameters.

<Tip>
Use the `equipment` values returned here as the `equipment` query parameter when calling `GET /api/v1/exercises` or `GET /api/v1/routines` to filter by equipment type.
</Tip>

<Note>
There are 28 equipment types in total. Fetching this endpoint is the recommended way to discover all valid equipment values before building filter UI or query logic.
</Note>

## Response schema

<ResponseField name="totalEquipments" type="number">
  Total number of equipment types (always `28`).
</ResponseField>

<ResponseField name="data" type="object[]">
  Array of equipment objects.

  <Expandable title="Equipment object properties" defaultOpen>
    <ResponseField name="equipment" type="string">
      Equipment name. Use this value as the `equipment` filter on `GET /api/v1/exercises` or `GET /api/v1/routines`.
    </ResponseField>
    <ResponseField name="imageUrl" type="string">
      Relative URL of the equipment's category image (`.webp` format).
    </ResponseField>
    <ResponseField name="exerciseCount" type="number">
      Number of exercises in the library using this equipment.
    </ResponseField>
  </Expandable>
</ResponseField>
