**`GET /api/v1/exercises`**

Returns a paginated list of exercises. Use query parameters to filter by body part, equipment, or target muscle, or to search by name and keyword.

## Query parameters

<ParamField query="page" type="number" default="1">
  Page number for pagination.
</ParamField>

<ParamField query="limit" type="number" default="10">
  Number of exercises to return per page.
</ParamField>

<ParamField query="bodypart" type="string">
  Filter by body part. Use values returned by `GET /api/v1/bodyParts`.

  Known values: `waist`, `chest`, `upper legs`, `lower legs`, `back`, `shoulders`, `upper arms`, `lower arms`, `neck`, `cardio`.
</ParamField>

<ParamField query="equipment" type="string">
  Filter by equipment type. Use values returned by `GET /api/v1/equipments`.

  Example values: `body weight`, `barbell`, `dumbbell`.
</ParamField>

<ParamField query="targetMuscle" type="string">
  Filter by target muscle group. Use values returned by `GET /api/v1/targetMuscles`.

  Example values: `abs`, `glutes`, `quads`, `hamstrings`, `pecs`.
</ParamField>

<ParamField query="search" type="string">
  Free-text search against exercise name and keywords.
</ParamField>

## Response schema

<ResponseField name="totalExercises" type="number">
  Total number of exercises matching the query.
</ResponseField>

<ResponseField name="totalPages" type="number">
  Total number of pages based on the requested `limit`.
</ResponseField>

<ResponseField name="data" type="object[]">
  Array of exercise objects.

  <Expandable title="Exercise object properties">
    <ResponseField name="id" type="string">
      Unique exercise identifier (zero-padded string, e.g. `"0001"`).
    </ResponseField>
    <ResponseField name="name" type="string">
      Lowercase exercise name.
    </ResponseField>
    <ResponseField name="title" type="string">
      Display-formatted exercise title.
    </ResponseField>
    <ResponseField name="target" type="string">
      Primary target muscle group.
    </ResponseField>
    <ResponseField name="muscles worked" type="string">
      Markdown-formatted list of muscles worked.
    </ResponseField>
    <ResponseField name="bodyPart" type="string">
      Body part category for the exercise.
    </ResponseField>
    <ResponseField name="equipment" type="string">
      Equipment required to perform the exercise.
    </ResponseField>
    <ResponseField name="blog" type="string">
      Markdown-formatted long-form description, instructions, and tips.
    </ResponseField>
    <ResponseField name="images" type="string[]">
      Array of image URLs showing the exercise. URLs are relative (no `https://` prefix).
    </ResponseField>
    <ResponseField name="gifUrl" type="string">
      URL of an animated GIF demonstrating the exercise. Relative URL.
    </ResponseField>
    <ResponseField name="videos" type="string[]">
      Array of YouTube video URLs related to the exercise.
    </ResponseField>
    <ResponseField name="keywords" type="string[]">
      Search keywords associated with the exercise.
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

### Basic request

<CodeGroup>

```bash cURL
curl "https://api.bodyworks.akshatjaiswal.me/api/v1/exercises?page=1&limit=10"
```

```javascript JavaScript
const response = await fetch(
  'https://api.bodyworks.akshatjaiswal.me/api/v1/exercises?page=1&limit=10'
);
const data = await response.json();
console.log(data);
```

</CodeGroup>

### Filtered request (waist body part + body weight equipment)

<CodeGroup>

```bash cURL
curl "https://api.bodyworks.akshatjaiswal.me/api/v1/exercises?bodypart=waist&equipment=body%20weight&limit=10"
```

```javascript JavaScript
const params = new URLSearchParams({
  bodypart: 'waist',
  equipment: 'body weight',
  limit: '10',
});

const response = await fetch(
  `https://api.bodyworks.akshatjaiswal.me/api/v1/exercises?${params}`
);
const data = await response.json();
console.log(data);
```

</CodeGroup>
