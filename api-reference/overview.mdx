## Base URL

All endpoints are served from:

```
https://api.bodyworks.akshatjaiswal.me/api/v1
```

The health check endpoint is at the root level:

```
https://api.bodyworks.akshatjaiswal.me/health
```

## Authentication

No authentication is required. All endpoints are public and can be called without any API key, token, or header.

## Response format

All responses are JSON. Successful responses return HTTP `200` with a JSON body. Collection endpoints wrap results in a `data` or `finalData` array and include pagination metadata. Single-resource endpoints return a `data` object.

## HTTP status codes

| Status | Meaning |
|--------|---------|
| `200` | Request succeeded. |
| `404` | The requested resource or endpoint does not exist. |
| `429` | Rate limit exceeded. Slow down and retry. |
| `500` | Internal server error. |

## Rate limits

| Endpoint | Limit |
|----------|-------|
| All endpoints (general) | Standard limits apply. |
| `GET /api/v1/routines` | 20 requests per 15-minute window. |

<Warning>
The routines endpoint enforces a stricter rate limit of **20 requests per 15 minutes**. Exceeding this limit returns a `429` response.
</Warning>

### Rate limit error responses

**General rate limit exceeded:**

```json
{
  "error": "Too many requests from this IP, please try again later."
}
```

**Routines rate limit exceeded:**

```json
{
  "error": "Rate limit exceeded for this endpoint."
}
```

## Error responses

**404 — Not found:**

```json
{
  "error": "Endpoint not found",
  "message": "The requested resource does not exist"
}
```

**500 — Internal server error:**

```json
{
  "error": "Internal server error",
  "message": "Something went wrong"
}
```

## Health check

Use this endpoint to verify the API is reachable and operational.

**`GET /health`**

<CodeGroup>

```bash cURL
curl https://api.bodyworks.akshatjaiswal.me/health
```

```javascript JavaScript
const response = await fetch('https://api.bodyworks.akshatjaiswal.me/health');
const data = await response.json();
console.log(data);
```

</CodeGroup>

**Example response:**

```json
{
  "status": "OK",
  "timestamp": "2024-01-15T10:30:00.000Z"
}
```
