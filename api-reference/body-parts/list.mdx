**`GET /api/v1/bodyParts`**

Returns all body part categories available in the exercise library. This endpoint takes no parameters.

<Tip>
Use the `bodyPart` values returned here as the `bodypart` query parameter when calling `GET /api/v1/exercises` to filter exercises by body part.
</Tip>

## Known body parts

There are 10 body part categories:

`waist` · `chest` · `upper legs` · `lower legs` · `back` · `shoulders` · `upper arms` · `lower arms` · `neck` · `cardio`

## Response schema

<ResponseField name="totalBodyParts" type="number">
  Total number of body part categories (always `10`).
</ResponseField>

<ResponseField name="data" type="object[]">
  Array of body part objects.

  <Expandable title="Body part object properties" defaultOpen>
    <ResponseField name="bodyPart" type="string">
      Body part name. Use this value as the `bodypart` filter on `GET /api/v1/exercises`.
    </ResponseField>
    <ResponseField name="imageUrl" type="string">
      Relative URL of the body part's category image (`.webp` format).
    </ResponseField>
    <ResponseField name="exerciseCount" type="number">
      Number of exercises in the library targeting this body part.
    </ResponseField>
  </Expandable>
</ResponseField>
